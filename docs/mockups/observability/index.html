Nexus · Observability mockups

Three connected views that turn alerts → logs → agent decisions into a single audit trail, plus a live wall that visualizes every running automation. Click each to open the mockup. Cards in the Hivemind link through to the drilldowns.

1Hivemind — live operations grid

A wall of every pipeline, agent, connector, schedule, and alert with live status. Active cards pulse, current verb ticks ("Reading 43 messages…", "Calling tool query_records…"), a footer ticker streams events. Click any card → run drilldown.

  • Lanes: Pipelines · Agents · Connectors · Schedules · Alerts
  • Per-card: status pulse, current activity, model badge, token counter
  • Top stats: running, failed, tokens/hr, records/min

2Run drilldown — pipeline failure

The view you land on when you click a failed alert or a failing pipeline card. Step timeline (left), filterable structured logs + dropped-record table (mid), metadata + suggested fixes + related entities (right).

  • Banner explains what failed, links back to the alert that opened the page
  • Tabs: Logs · Records · Stats · Lineage · Raw payload
  • Per-step funnel (in → out → dropped) + collapsible Python stack trace
  • "Suggested fixes" pane with one-click open in builder

3Agent decision trace

Replay an agent run as a transcript: user prompt → reasoning → tool call → tool result → next reasoning → final answer. Each iteration is its own block, streaming runs show a live caret, human-gate checkpoints render inline.

  • Left rail: every iteration in the reasoning loop
  • Mid: alternating thinking / tool / tool-result / assistant blocks
  • Token cost bar (in / out / cache) and "compare to previous run" panel
  • Toggles: hide reasoning · show only failed iterations
